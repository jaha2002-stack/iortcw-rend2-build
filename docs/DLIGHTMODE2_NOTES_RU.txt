# Техническое объяснение

В rend2 documentation r_dlightMode описан так:
0 = fake RTCW brightening
1 = actual lighting, no shadows
2 = light and shadows, broken

В текущем коде r_dlightMode 2 использует ForwardDlight() в tr_shade.c.
Этот path уже ближе к нужному advanced lighting, потому что он работает через lightallShader и биндит:
- diffuse map
- normal map
- specular map
- light origin/radius
- specular/normal scale

Проблемная часть: shadow cubemap binding для r_dlightMode >= 2.
В этом экспериментальном патче она заменена на safe white texture.
Это не даёт настоящих теней, но убирает наиболее вероятный источник broken shadow artifacts.

Следующий настоящий этап:
- реализовать корректную генерацию cubemap depth shadow maps для selected dynamic lights
- добавить PCF
- bias
- limit по количеству shadow-casting dlights
- корректную интеграцию с lightallShader

Этот kit — промежуточный практический шаг, не финальная система point-light shadows.
