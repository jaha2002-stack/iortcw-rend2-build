# Limitations

Я не буду скрывать ограничения:

- Этот kit не гарантирует идеальные настоящие тени на всех факелах.
- Если конкретные факелы на карте не являются renderer dynamic lights, они не смогут cast shadows.
- Если штатная генерация tr.shadowCubemaps не заполняется корректно для этих dlights, видимой тени не будет.
- Полная реализация потребует переписать backend pass:
  выбрать lights -> render 6 depth faces -> fill cubemap -> sample with PCF/bias.

Что этот kit уже делает:
- создаёт отдельный mode3
- добавляет cvar-интерфейс
- ограничивает количество lights
- убирает broken глобальный mode2
- включает shadow path только контролируемо
